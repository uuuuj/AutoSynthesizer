# 로직 변경 이력 (2026-03-28)

## 변경 1: 컬럼 설명(Description) 입력 UI 추가

### 변경 위치

- `_build_ui()` — ③ 가짜 데이터 입력과 확정 버튼 사이에 ③-1 섹션 추가
- `_build_desc_widgets()` — 신규 메서드
- `_analyze_file()` — `_build_desc_widgets()` 호출 추가
- `_refresh_all()` — `col_desc_entries`, `desc_inner` 초기화 추가

### 변경 내용

- 각 컬럼에 대한 설명(description)을 사용자가 직접 입력할 수 있는 UI 섹션 신설
- 변환된 컬럼명 옆에 Entry 위젯을 배치하여 컬럼의 성격·용도를 기술하도록 안내
- placeholder 텍스트로 입력 가이드 제공 (포커스 진입 시 제거, 이탈 시 복원)
- placeholder 안내: "원본 컬럼명을 직접 언급하지 말고, 컬럼의 성격·용도만 기술"

### 인스턴스 변수

- `self.col_desc_entries` — `{원본 컬럼명: Entry 위젯}` 딕셔너리 추가

## 변경 2: 출력 JSON 파일 구조 변경

### 변경 위치

- `_do_synth()` 메서드 내 파일 저장 부분

### 변경 내용 (Before → After)

**삭제:**
- `_변환키.json` — 원본 컬럼, 현재 컬럼, 컬럼 리네임, 값 매핑을 모두 포함하던 복원키 파일 제거

**신규 파일 1: `_컬럼매핑.json`**
- 원본 컬럼명 → 변환 컬럼명 매핑만 저장
- 구조: `{ "generated_at": "...", "column_mapping": { "원본A": "변환A", ... } }`
- 용도: 필요 시 컬럼을 원래 이름으로 복구하기 위한 매핑 정보

**변경 파일: `_description.json`**
- 기존: `synthesize_text_columns()` 함수의 `desc_map` (자동 생성된 매핑 방법 기록) 저장
- 변경: 사용자가 ③-1 섹션에서 직접 입력한 컬럼 설명 저장
- 구조: `{ "변환컬럼A": "사용자 입력 설명", ... }`
- 용도: 외부망에서 변환된 컬럼명만 보고도 해당 컬럼의 성격을 이해할 수 있도록 하는 메타데이터

### 최종 출력 파일 목록

| 파일 | 내용 |
|------|------|
| `.xlsx` | 합성된 데이터 |
| `_컬럼매핑.json` | 원본 컬럼 → 변환 컬럼 매핑 |
| `_description.json` | 변환 컬럼별 사용자 입력 설명 |
| `_품질리포트.json` | 합성 품질 검증 결과 |
| `.parquet` 또는 `.csv` | 합성 데이터 (Parquet 우선, 실패 시 CSV) |

## 변경 3: 기타 UI 문구 수정

- 저장 설정 안내 레이블: `_변환키.json` → `_컬럼매핑.json`
- 단계 힌트: "컬럼명·데이터·설명을 확인 후" 로 변경
- 확정 버튼 안내: "컬럼명·가짜 데이터·컬럼 설명을 확인한 뒤" 로 변경
- 파일 상단 docstring: ③-1 컬럼별 설명 입력 단계 추가, ⑥ 복원 단계 제거
