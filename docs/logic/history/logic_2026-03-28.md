# 로직 변경 이력 (2026-03-28)

## 변경 1: 컬럼 설명(Description) 입력을 ②-1 컬럼명 변경 테이블에 통합

### 변경 위치

- `_build_ui()` — ②-1 섹션 제목 변경: "컬럼명 변경 + 설명 입력"
- `_build_rename_widgets()` — 각 행에 설명 입력 Entry 열 추가
- `_refresh_all()` — `col_desc_entries` 초기화 추가
- `_analyze_file()` — `col_desc_entries` 초기화 추가

### 변경 내용

- ②-1 컬럼명 변경 테이블에 **"설명"** 열을 추가하여 한 행에서 컬럼명 변경과 설명 입력을 동시 수행
- 별도 ③-1 섹션은 제거 (초기 구현에서 변경)
- 설명 입력은 **선택 사항** — 빈칸이어도 변환 확정 및 실행 가능
- placeholder 텍스트: `"변환 컬럼만 보고 원본을 연상할 수 있도록 성격·용도를 서술"`
  - 포커스 진입 시 placeholder 제거, 포커스 이탈 시 비어있으면 복원
  - placeholder 상태로 남은 항목은 빈 문자열(`""`)로 저장

### 테이블 열 구성

| No. | 기존 컬럼명 | → | 새 컬럼명 | 타입 | 설명 |
|-----|------------|---|----------|------|------|

### 인스턴스 변수

- `self.col_desc_entries` — `{원본 컬럼명: Entry 위젯}` 딕셔너리
- `_build_rename_widgets()` 내에서 생성, `_refresh_all()` / `_analyze_file()` 에서 초기화

## 변경 2: 출력 JSON 파일 구조 변경

### 변경 위치

- `_do_synth()` 메서드 내 파일 저장 부분

### 변경 내용 (Before → After)

**삭제:**
- `_변환키.json` — 원본 컬럼, 현재 컬럼, 컬럼 리네임, 값 매핑을 모두 포함하던 복원키 파일 제거

**신규 파일: `_컬럼매핑.json`**
- 원본 컬럼명 → 변환 컬럼명 매핑만 저장
- 구조: `{ "generated_at": "...", "column_mapping": { "원본A": "변환A", ... } }`
- 용도: 필요 시 컬럼을 원래 이름으로 복구하기 위한 매핑 정보

**변경 파일: `_description.json`**
- 기존: `synthesize_text_columns()` 함수의 `desc_map` (자동 생성된 매핑 방법 기록) 저장
- 변경: 사용자가 ②-1 테이블에서 직접 입력한 컬럼 설명 저장
- 구조: `{ "변환컬럼A": "사용자 입력 설명", ... }`
- 용도: 외부망에서 변환된 컬럼명만 보고도 해당 컬럼의 성격을 이해할 수 있도록 하는 메타데이터
- 미입력 항목은 빈 문자열로 저장

### 최종 출력 파일 목록

| 파일 | 내용 |
|------|------|
| `.xlsx` | 합성된 데이터 |
| `_컬럼매핑.json` | 원본 컬럼 → 변환 컬럼 매핑 |
| `_description.json` | 변환 컬럼별 사용자 입력 설명 |
| `_품질리포트.json` | 합성 품질 검증 결과 |
| `.parquet` 또는 `.csv` | 합성 데이터 (Parquet 우선, 실패 시 CSV) |
