# 로직 변경 이력 (2026-04-06)

## 변경 1: ③ 가짜 데이터 편집/미리보기 UI 제거

### 변경 위치

- `_build_ui()` — ③ 가짜 데이터 입력 섹션(sec3) 전체 제거
- `_build_input_widgets()` — 메서드 삭제
- `_auto_fill_column()` — 메서드 삭제
- `_auto_fill_all()` — 메서드 삭제
- `_validate_mappings()` — 메서드 삭제
- `_collect_mappings()` — 메서드 삭제

### 변경 내용

- 사용자가 가짜 데이터(원본값 → 가짜값)를 편집·미리보기하는 UI 섹션을 완전히 제거
- 가짜 데이터 매핑은 기존 자동 채우기 로직과 동일하게 **내부에서 자동 생성**
- 결과물은 **엑셀 파일에서만 확인**하도록 변경
- 확정 버튼(✅ 변환 계획 확정)은 **유지** — 컬럼명·설명 확인 후 확정하는 용도

### 제거된 UI 요소

| 요소 | 설명 |
|------|------|
| `sec3` (LabelFrame) | "③ 가짜 데이터 입력 (원본값 → 가짜값)" 섹션 |
| `self.input_inner` (Frame) | 가짜 데이터 입력 위젯 컨테이너 |
| `self.auto_all_btn` (Button) | "🔄 전체 자동 채우기" 버튼 |
| 컬럼별 Entry 위젯들 | 원본값 → 가짜값 1:1 입력 필드 |

### 제거된 인스턴스 변수

- `self.col_entry_map` — `{컬럼명: [(원본값, Entry위젯, 고정값), ...]}` 딕셔너리

## 변경 2: `_generate_auto_mappings()` 메서드 신규 추가

### 변경 위치

- `_collect_mappings()` 자리에 `_generate_auto_mappings()` 추가

### 변경 내용

- UI 위젯 없이 프로그래밍 방식으로 가짜 데이터 매핑을 자동 생성
- 기존 `_auto_fill_column()` + `_collect_mappings()` 의 로직을 통합
- ID형 컬럼: 원본 접두사와 다른 접두사로 순번 ID 생성
- 한글 이름 패턴: `generate_fake_persons()` 사용
- 회사명 키워드: `generate_fake_companies()` 사용
- 그 외: `generate_auto_codes()` 사용 (변환된 컬럼명 기반)

### 호출 위치

- `_do_synth()` 내 `[1/8] 매핑 수집` 단계에서 `self._generate_auto_mappings()` 호출
