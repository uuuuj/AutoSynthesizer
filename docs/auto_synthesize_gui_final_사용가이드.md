# 정형 데이터 마스킹 프로그램 사용 가이드

> **파일:** `auto_synthesize_gui_final.py`
> **목적:** 엑셀 정형 데이터를 읽어 컬럼명 변환, 값 마스킹, 수치/날짜 합성을 수행하여 외부망에서 안전하게 사용할 수 있는 합성 데이터를 생성한다.

## 2. GUI 구성 (단계별 사용법)

프로그램은 상단 단계 표시 바에 현재 진행 상태가 표시되며, 순서대로 진행한다.

```
❶ 파일 선택  ➤  ❷ 파일 분석  ➤  ❸ 확인/수정  ➤  ❹ 변환 확정  ➤  ❺ 실행
```

---

### ❶ 엑셀 파일 선택

1. **찾아보기** 버튼으로 엑셀 파일(`.xlsx`, `.xls`, `.xlsm`)을 선택한다.
2. 시트가 여러 개인 경우 **시트 선택** 드롭다운에서 원하는 시트를 선택한다.
3. **생성 행 수**를 입력한다. 비워두면 원본과 동일한 행 수로 생성된다.
4. **📊 파일 분석** 버튼을 클릭한다.

> **행 수 제한:** 원본 데이터가 500행을 초과하면 상위 500행만 사용된다.

---

### ❷ 파일 분석 결과 확인

파일 분석이 완료되면 상단에 요약 정보가 표시된다.

```
파일: data.xlsx  |  시트: Sheet1  |  100행 x 8열  (수치 3, 날짜 1, 범주 4)
📐수치 매출액  |  📅날짜 계약일  |  🏷️범주 부서명  |  ...
```

각 컬럼은 자동으로 3가지 타입 중 하나로 분류된다.

| 아이콘 | 타입 | 설명 | 처리 방식 |
|--------|------|------|----------|
| 📐 | numerical | 숫자 데이터 | Gaussian Copula로 상관관계 보존 합성 |
| 📅 | datetime | 날짜/시간 데이터 | 날짜 분포 기반 합성 |
| 🏷️ | categorical | 문자열/범주 데이터 | 사용자 입력 매핑으로 1:1 치환 |

---

### ②-1 컬럼명 변경 + 설명 입력

파일 분석 후 컬럼명 변경 테이블이 자동 생성된다. 각 행에는 다음 항목이 있다.

| 열 | 설명 |
|----|------|
| **기존 컬럼명** | 원본 엑셀 파일의 컬럼명 (읽기 전용) |
| **새 컬럼명** | 변환할 컬럼명 입력. 비워두면 원본 이름 유지 |
| **타입** | 자동 감지된 컬럼 타입 |
| **설명** | 컬럼에 대한 설명 입력 **(선택 사항)** |

#### 컬럼명 자동 채우기

- 파일 분석 시 타입별로 자동 이름이 입력된다: `Text1`, `Text2`, `Num1`, `Num2`, `Date1` 등
- 원하는 이름이 있으면 직접 수정한다.
- **✅ 컬럼명 적용** 버튼으로 중간에 적용할 수도 있고, 변환 확정 시 자동 적용된다.

#### 설명 입력 가이드

설명은 **외부망에서 변환된 컬럼명(`Text1`, `Num3` 등)만 보고도 어떤 데이터인지 이해할 수 있도록** 작성하는 메타데이터이다.

**작성 규칙:**
- 원본 컬럼명을 **직접 언급하지 않는다** (예: ~~"부서명 컬럼"~~ → "조직 단위를 나타내는 범주")
- 변환된 컬럼을 보았을 때 원본 컬럼의 **성격과 용도를 연상**할 수 있도록 서술한다.
- 입력하지 않아도 진행에 문제 없다 (선택 사항).

**작성 예시:**

| 원본 컬럼 | 변환 컬럼 | 설명 (좋은 예) |
|----------|----------|---------------|
| 담당자 | Text1 | 작업 수행 인원의 이름 |
| 공정코드 | Text2 | 생산 공정을 구분하는 식별 코드 |
| 작업시간 | Num1 | 단위 작업당 소요 시간 (시간 단위) |
| 계약일 | Date1 | 거래가 체결된 일자 |

---

### ❸ 가짜 데이터 1:1 입력

범주형(categorical) 컬럼만 표시되며, 각 고유값에 대한 가짜값을 입력한다.

```
📋 부서명 (5개)
──────────────
  영업부    → [Alpha Corp  ]
  개발부    → [Beta Group   ]
  인사부    → [Gamma Inc    ]
```

#### 자동 채우기 기능

| 버튼 | 동작 |
|------|------|
| **🔄 전체 자동 채우기** | 모든 범주 컬럼의 빈 항목을 한번에 자동 생성 |
| **🔄 자동 채우기** (컬럼별) | 해당 컬럼의 빈 항목만 자동 생성 |

자동 채우기 로직:

| 원본 값 패턴 | 생성 방식 |
|-------------|----------|
| 한글 2~4글자 이름 | 가짜 한국 이름 생성 (김민준, 이서연 등) |
| 고객/발주/업체 관련 컬럼 | 가짜 회사명 생성 (Alpha Corp, Beta Group 등) |
| ID형 데이터 (접두사+숫자) | 다른 접두사로 고유 ID 재생성 |
| 기타 | `{변환컬럼명}_001`, `{변환컬럼명}_002` 형식 코드 생성 |

> 자동 채우기 후에도 원하는 값으로 직접 수정할 수 있다.

---

### ❹ 변환 확정

모든 입력을 확인한 뒤 **✅ 변환 계획 확정** 버튼을 클릭한다.

- 가짜값 중복 경고: 같은 컬럼 내에서 서로 다른 원본값에 동일한 가짜값이 할당된 경우 경고가 표시된다.
- 확정 후에는 **▶ 합성 데이터 생성 실행** 버튼이 활성화된다.

---

### ④ 저장 설정

| 항목 | 설명 |
|------|------|
| **저장 경로** | 결과물이 저장될 폴더. 기본값은 원본 파일과 같은 폴더 |
| **파일 이름** | 기본값: `{원본파일명}_합성데이터`. 확장자는 자동 부여 |

---

### ❺ 실행

**▶ 합성 데이터 생성 실행** 버튼을 클릭하면 8단계 파이프라인이 진행된다.

| 단계 | 작업 |
|------|------|
| [1/8] 매핑 수집 | 입력된 가짜값 매핑 정리 |
| [2/8] 문자열 합성 | 범주형 컬럼 값 치환 (null 보존) |
| [3/8] 종속성 분석 | 컬럼 간 함수 종속 관계 감지 |
| [4/8] 상관관계 | 수치 컬럼 간 상관관계 분석 |
| [5/8] 제약조건 | 양수, 0~100 범위, 대소 관계 등 감지 |
| [6/8] 수치 합성 | Gaussian Copula 기반 수치/날짜 합성 |
| [7/8] 행 조합 | 범주 분포 보존 가중 샘플링 |
| [8/8] 품질 검증 | 원본 대비 통계적 유사도 검증 |

실행 로그가 실시간으로 표시되며, 프로그레스바로 진행률을 확인할 수 있다.

## 3. 출력 파일

실행 완료 시 다음 파일들이 저장 경로에 생성된다.

| 파일 | 확장자 | 내용 |
|------|--------|------|
| **합성 데이터** | `.xlsx` | 마스킹 및 합성이 완료된 엑셀 데이터 |
| **컬럼매핑** | `_컬럼매핑.json` | 원본 컬럼명 → 변환 컬럼명 매핑 (내부 보관용) |
| **Description** | `_description.json` | 변환 컬럼별 사용자 입력 설명 (외부망 참고용) |
| **품질리포트** | `_품질리포트.json` | 합성 품질 점수, 컬럼별 유사도 등 |
| **Parquet/CSV** | `.parquet` 또는 `.csv` | 합성 데이터 (Parquet 우선, 실패 시 CSV) |

### 출력 파일 상세

#### `_컬럼매핑.json`

```json
{
  "generated_at": "2026-03-28T14:30:00",
  "column_mapping": {
    "부서명": "Text1",
    "담당자": "Text2",
    "매출액": "Num1",
    "계약일": "Date1"
  }
}
```

- **용도:** 내부망에서 보관. 변환된 컬럼명을 원본 컬럼명으로 복구할 때 참고.
- **외부 반출 금지** — 원본 컬럼명이 포함되어 있으므로 내부에서만 사용.

#### `_description.json`

```json
{
  "Text1": "조직 단위를 나타내는 범주",
  "Text2": "작업 수행 인원의 이름",
  "Num1": "거래 금액 (원 단위)",
  "Date1": "거래가 체결된 일자"
}
```

- **용도:** 외부망에서 변환된 컬럼명과 함께 참고하여 데이터 이해.
- **원본 컬럼명이 포함되지 않으므로 외부 반출 가능.**
- 미입력 항목은 빈 문자열로 저장된다.

#### `_품질리포트.json`

```json
{
  "generated_at": "2026-03-28T14:30:00",
  "source_file": "data.xlsx",
  "sheet_name": "Sheet1",
  "original_rows": 100,
  "synthetic_rows": 100,
  "column_types": { "Num1": "numerical", "Text1": "categorical", ... },
  "quality_overall": 0.92,
  "quality_by_col": { "Num1": 0.95, "Text1": 0.88, ... },
  "strong_correlations": [...]
}
```

## 4. 합성 알고리즘 상세

### 4.1 컬럼 타입 자동 감지

`auto_detect_column_type()` 함수가 각 컬럼을 분석하여 타입을 결정한다.

| 판정 순서 | 조건 | 결과 |
|----------|------|------|
| 1 | 날짜형으로 파싱 성공률 > 50% | `datetime` |
| 2 | 숫자형 dtype | `numerical` |
| 3 | 그 외 | `categorical` |

### 4.2 수치 합성 — Gaussian Copula

1. 각 수치 컬럼을 정규분포로 변환 (Rank → Normal)
2. 변환된 공간에서 다변량 정규분포 피팅
3. 상관 행렬을 보존하며 새 데이터 샘플링
4. 역변환으로 원래 분포 복원
5. 제약조건 적용 (양수, 범위, 대소 관계 등)
6. 원본과 동일한 비율로 null 재삽입

### 4.3 날짜 합성

1. 기준 날짜(최소값)로부터의 일수 차이 계산
2. 일수 분포를 수치 컬럼과 동일한 Copula 파이프라인으로 합성
3. 기준 날짜에 합성된 일수를 더하여 날짜 복원

### 4.4 함수 종속성 보존

- 컬럼 간 1:1 결정 관계(예: 부서코드 → 부서명)를 자동 감지
- 합성 후 종속 관계가 유지되도록 매핑 테이블 기반 복원

### 4.5 ID 컬럼 처리

- 고유값 비율 90% 이상인 컬럼을 ID형으로 판정
- 접두사를 원본과 다른 것으로 변경하여 고유 ID 재생성
- 예: `SHI001` → `XRK00001`

### 4.6 품질 검증

| 타입 | 검증 방법 |
|------|----------|
| 수치 | KS 검정 (분포 유사도) |
| 범주 | 빈도 분포 비교 |
| 날짜 | 타임스탬프 KS 검정 |

종합 품질 = 전체 컬럼 점수의 평균

## 5. 주의사항

1. **원본 파일은 수정되지 않는다.** 합성 결과는 항상 새 파일로 저장된다.
2. **500행 제한:** 원본이 500행을 초과하면 상위 500행만 분석에 사용된다.
3. **`_컬럼매핑.json`은 외부 반출하면 안 된다.** 원본 컬럼명이 포함되어 있다.
4. **`_description.json`과 합성 `.xlsx`만 외부망으로 전달한다.**
5. 실행 중에는 전체 UI가 비활성화되며, 완료 시 자동으로 복원된다.
6. **가짜값 중복 주의:** 같은 컬럼 내에서 서로 다른 원본값에 동일한 가짜값을 매핑하면 데이터 구분이 불가능해진다. 경고 메시지가 표시되면 수정을 권장한다.

## 8. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| xlwings 로드 실패 | Excel 미설치 또는 COM 오류 | openpyxl로 자동 폴백됨 (정상) |
| DRM 파일 읽기 실패 | openpyxl은 DRM 미지원 | xlwings + Excel 설치 필요 |
| 품질 점수가 낮음 | 원본 데이터 행 수가 적거나 분포가 복잡 | 생성 행 수를 늘리거나 원본 데이터 확인 |
| Parquet 저장 실패 | pyarrow/fastparquet 미설치 | CSV로 자동 폴백됨 (정상) |
| 프로그레스바 멈춤 | 합성 연산 중 (정상) | 실행 로그가 계속 갱신되면 정상 동작 중 |
